---
title: "Render deployment"
description: "Operate Lumina on its existing Render web service."
---

Lumina deploys as one NestJS web service. REST, Swagger, health probes, and MCP Streamable HTTP all share Render's assigned `PORT`.

## Service settings

```text
Build command: npm ci && npm run build
Start command: npm run start:prod
Health check path: /health/ready
```

Set `MCP_HTTP_ENABLED=false`; the separate legacy SSE listener is unnecessary on Render because `/mcp` is handled by the primary NestJS server.

Enable marketplace payment access with `X402_ENABLED=true`, `X402_NETWORK=eip155:196`, `X402_PRICE_USD=$0.01`, `X402_PAY_TO_ADDRESS`, and `X402_RESOURCE_URL=https://lumina-e3vi.onrender.com/mcp`. The existing OKX API credentials are also used by the payment facilitator.

## Release verification

```bash
curl https://lumina-e3vi.onrender.com/health
curl https://lumina-e3vi.onrender.com/health/ready
curl -I https://lumina-e3vi.onrender.com/docs
```

Then verify an authenticated REST translation, the unauthenticated MCP `402` challenge, authenticated MCP initialization, a signed GitHub ping, and bearer-protected metrics.
