Lumina exposes MCP Streamable HTTP at:

```text
https://lumina-e3vi.onrender.com/mcp
```

Trusted consumers can send a bearer key with every MCP request.

```json
{
  "mcpServers": {
    "lumina": {
      "type": "streamable-http",
      "url": "https://lumina-e3vi.onrender.com/mcp",
      "headers": {
        "Authorization": "Bearer ${LUMINA_API_KEY}"
      }
    }
  }
}
```

## OKX.AI payment access

Marketplace callers without a bearer key receive a standard x402 v2 `402 Payment Required` response. The `PAYMENT-REQUIRED` header contains the payment challenge for X Layer. After payment, the caller replays the request and receives the MCP response with settlement proof.

Bearer authentication remains available for existing SDK and operator integrations. A verified x402 payment grants access only to MCP and cannot be used for REST or administration routes.

Production configuration:

```bash
X402_ENABLED=true
X402_NETWORK=eip155:196
X402_PRICE_USD=$0.01
X402_PAY_TO_ADDRESS=0xYourReceivingAddress
X402_RESOURCE_URL=https://lumina-e3vi.onrender.com/mcp
```

Lumina uses the official OKX payment packages for challenge generation, verification, replay protection, and settlement.

## Tools

### `translate_text`

Translates dApp copy while protecting structural tokens. Inputs: `text` and `targetLanguage`.

### `decode_error`

Converts an OKX OnchainOS or EVM error code into a localized actionable explanation. Inputs: `code` and optional `targetLanguage`.

### `glossary_lookup`

Returns a Web3 glossary entry and its verified localized mappings. Input: `term`.

<Warning>
  Do not expose the bearer key in a public frontend. Configure it in the agent platform's encrypted secrets or server environment.
</Warning>
